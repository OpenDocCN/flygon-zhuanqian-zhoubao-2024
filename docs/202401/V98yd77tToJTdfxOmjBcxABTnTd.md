# 项目简介

大家好，我是生财航海志愿者-David，也是服务生财同城聚会50+场的志愿者。

作为航海志愿者，一个重要工作就是航海微信群成员的服务工作。拓展开来，对于一个社群运营，如何低成本高效服务好微信群成员也是每个社群运营者需要考虑的问题。

本次航海我参加了RPA航海志愿者服务，通过RPA工具极大的提高了我服务效率。之前服务一个40人规模的微信群活动，我需要花1个小时完成加微信，建群，邀请进群这些动作；使用RPA提效工具后，我只需要花2分钟配置好参数和话术，让RPA机器人自动运行就可以。在RPA机器人执行完成后，我检查执行结果然后再做相应动作即可。另外，因为每个流程模块编写好后，我也不用担心自己手动操作会有遗漏。

#### 登录工作微信

为了避免微信因为频繁添加好友被封禁，可以分时段分批添加，一次最多不要连续添加超过10个微信号。

可以通过调整【参数表】中的【起始行】和【结束行】来设定每次添加的微信数量。

同时，把【延时】项的数值调到100000以上，这里延时是按毫秒计算，因此100000表示需延时100000毫秒=100秒。

【添加朋友申请】项用来设定添加微信时使用的申请信息

【标签】项用来给添加的微信号设定标签，便于管理

运行结束后可以在【输入表】中查看添加结果。

如果【上船进度】列中显示【此微信号不存在】，则需要我们使用手机号手动再次添加。

# 项目总结

## 业务流程SOP是前提

因为志愿者工作流程SOP已经很详细，所以我在实现RPA提效工具时可以只关注如何把SOP中的操作用RPA机器人自动完成。根据我的软件开发经验，业务人员对工作流程更清楚，但如果要把工作流程交给软件开发人员来实现时，软件开发人员常常不能准确理解业务流程逻辑，这样沟通成本很高，而且前期沟通不到位，后期再来改动时费时费力还可能导致项目失败。

业务人员如果自己能按照现有业务流程快速实现RPA机器人原型，完成手上工作，就会很配合实施业务流程SOP。

## RPA对非技术人员友好

整个项目实施过程中，没有涉及到编程语言的学习，所有操作都是图形化模块拖放完成的。其难度不超过航海手册的示例。而我们收获了一个功能强大的应用模块，包括：自动添加微信好友，微信消息自动发送，自动提醒，自动同步状态到网站。

## 能快速解决问题是关键

RPA能快速实现软件操作的自动化，只要你会用某个软件，就可以使用RPA工具来做这个软件的重复工作。

例如，对40个微信好友单独发送提醒消息，如果你是技术小白不会编程，你可能会找到像易歪歪这样的提效工具，但是还是需要一个个微信好友点开来发送。用RPA就比较直观了，按照平常我们给微信好友发消息的步骤配置好RPA机器人，然后让机器人自动运行就可以了。一次搭建，重复使用。

## 模块可复制可重用

在完成了生财航海微信群RPA机器人制作同时，我在生财线下同城聚会活动中也使用了这个工具，服务效率也同样十倍的提升，具体可以参考：
