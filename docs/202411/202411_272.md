## (精华帖)(190 赞)YouTube Shorts 项目崛起：小作坊的机遇与挑战

作者： 老馬🐎内容出海

最近一段时间很忙，忙着加号没时间刷生财，今天看到亦仁大佬的超级标，瞬间就坐不住了。 作为沉浸在 YouTube
Shorts 两年的小作坊来说，这个项目可终于被别人看到了。 又有点害怕，又有点兴奋，兴奋的是不孤单了，害怕的是，会不会被冲爆~~
Shorts 去年还好，主要光环都在 Tt 中视频那边，从今年年初开始，Shorts 就开始发力了。
单价从最早的 0.08rpm，到现在 0.3，这样的涨幅可不是一点半点的增长。
也算幸运，去年 tiktok 中视频没怎么赚到钱，今年搞 Shorts 算是吃到了一波红利。
我们做了 600 个频道，开通收益的有 20 多个，大致都沉浸在匿名频道，也就是不露脸的视频上耕耘。 主要做解说类，AI 类，萌宠类，这三个赛道上。
江鲤大佬早上分享了关于 rpa 自动化采集的东西对我来说很受用，因为，我们在做这些的时候，免不了的就是要学习对标；
我们需要找到对标，扩展，筛选，反推，然后批量复制，自动化。
之前都是手动整理对标，手机去刷，从没想过可以通过标签这些来持续拓展，然后去重，感谢大佬提供的新思路。 如果江鲤大佬早点分享，
我可能月初就没那么煎熬了，为了采集这些东西，用了好多时间学习这些代码。
言归正传，江鲤大佬使用的是影刀 rpa 采集，而我用的是 selenium 的方式，但是这个方法对我来说还是很麻烦，又要对接指纹浏览器又要找到一些定位元素，毕竟自己每日采集的数据量不大，我们是每天在跟踪一些我们筛选后的对标，所以我换了另一种方法，使用的正是江鲤大佬所提到的油管官方的 api，好处是真方便，缺点是，每天只有一万的额度。但这个方法对于一些采集量不大，并且想直接能够上手用的新手入局来说，是不错的选择。
(后面我会把 selenium 和 api 的调用方式都分享出来)
我在调用官方 api 的时候，它可以通过输入 YouTube 频道名称（例如@hublotbaggins），自动获取该频道所有视频的标题、发布时间、播放量，然后在通过数据筛选，抽取近期百万播放的视频链接以及相关信息。
