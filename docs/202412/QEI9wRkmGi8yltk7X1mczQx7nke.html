二、如何正确使用工具

对于刚入门的小伙伴们，由于过去和“产品开发”相距甚远，首要面临的问题是，缺乏编程及编程基础知识，因此导致了我们容易习惯性的将Cursor理解为无所不能的编程机器人一样，（这是非常正常且合理的现象）。

即：我们只需要通过自然语言对Cursor提出需求，能让它听懂，它就能听懂人类的指令并产出结果。实际上在经过一段时间对Cursor的使用观察发现，想做到这一点一点都不容易。后面我会简单示范一下，通过不同的“自然语言风格”对Cursor提出指令，Cursor的理解有何变化，并会做出什么样的反应。

需要注意的是，在使用Cursor编程开发工具的过程中，偶尔会刷到一些大佬 ，在网络社交平台发布的诸如“用Cursor5分钟开发一个网站”、“通过Cursor半个小时开发一款APP”之类的帖子。

当我们看到这些显性结果的同时，容易忽略这些开发者前期付出的隐性时间成本和思考积累，从而对实现Cursor智能编程产生一种比较偏离的认知和预期。因此，在个人实际操作过程中，就容易产生动作变形、结果差于预期。从而陷入焦虑和焦躁当中。

不过，既然提出了问题，咱们一定会给出解决方案。如何正确使用工具，主要包括两部分：Gemini/ChatGPT+Kimi、另一个是快速且正确的使用Cursor。

为什么开发应用除了用Cursor，还需要借助辅助工具呢？

通过测试和实践我发现，虽然Cursor在代码生成和局部优化方面表现的挺不错的，但对于复杂项目的层次规划、架构设计、以及理解复杂的错误信息等方面，可能存在一定的局限性。而Gemini、ChatGPT在项目工程理解和自然语言对话方面的优势就出来了。因此，咱就可以借助这些辅助工具来补充Cursor的不足，另外也可以更少的；浪费Cursor的请求次数资源。

2.如何快速且正确的使用Cursor

2.1 首先要对Cursor的个性以及能力有所了解。

Cursor是经常会出错、也是会看人下菜碟的。什么意思呢？其实就是它给出的代码和思路仅仅只能是一个大致的参考，虽然大多数时候是正确的，但很多时候也会错误百出。因此，在面临复杂项目的时候，如果没有开发经验，往往会陷入不断调试、不断出错的怪圈。所以最好的方式应该是在做项目的过程中学习编程语言、逐步掌握开发的流程和各种相关工具。

那如何解决这类问题？我的方法主要分为三部分：

2.2 cursor的常用功能

比如，下面这几项都是比较需要常用到的功能：

-@Codebase:

在chat输入框手动输入@，然后出现下拉框，就可以选择Codebase。通过@Codebase，可以直到找到要使用的最重要的代码片段，并基于这些代码，让cursor进行分析和优化。

-@Files:

通过这个功能、我们让Cursor 来找到我们希望它查看的代码文件，比如我们希望它检查一下index.html这个代码文件中的代码逻辑和代码没有错误或者需要完善的，那么我们就可以在聊天框中@Files。

-@Folders:

通过@Folders我们可以让cursor选中对应的文件夹，并回顾对应的文件夹，从而让它快速遍历整个文件夹的内容，从而发现问题，如下：

-@Docs：

比如我们现在正在开发一款ios APP，我们希望这款APP的界面设计效果实现Apple的简洁效果，接下来就可以通过@Docs调出这个功能，然后点击Add new doc，从而粘贴所需文档的 URL 。

比如，我在开发APP的过程中就引用了Apple官网上的一个Apple UI链接 在苹果官网专门有这个页面，复制链接过来就可以了：https://developer.apple.com/cn/documentation/swiftui/ （如下图操作）

-command+K（ctrl+k)

我们可以在cursor里打开左侧的一个文件，然后在对应的代码上面，打开command+k，然后输入：帮我注释下面这段代码，点击生成，cursor就会帮我们把每行代码生成注释，这样对我们理解代码也是一种不错的学习方法。

另外也可以在编辑器窗口中也可以生成新代码或编辑现有代码。

-chat和composor

如果追求速度快的话，建议使用composor，因为我们可以实现Accept all，不过这个功能的问题在于，替换代码的时候，经常会出错。

使用chat的话，可能需要我们多动手进行代码替换、手动找文件。但是，只要耐心去发掘它的功能，它的作用一点不比composor差。对这块儿感兴趣的小伙伴欢迎多交流！

-design.md文件

主要用于详细描述项目的设计方面，包括但不限于系统的架构、组件设计、用户界面设计、用户体验设计等。 它可以包含项目的目录结构、技术要点、设计决策的解释、以及任何与设计相关的图表或原型。design.md 有助于cursor理解项目的视觉和功能设计，确保设计意图在开发过程中得到准确实现。

-README.md

这个文件相当于项目的“自述文件”，一般放在项目根目录下。主要用于向用户或开发者介绍项目，比如我们把自己的项目代码放到GIthub上面的，有人访问我们项目的源代码，首先就会查看这个文件。这个文件通常介绍项目概括、安装指南、如何运行项目、使用说明、贡献指南、版权和许可信息等。此外，README.md 也能包含项目的背景信息、目的、主要功能、技术栈、依赖关系、以及如何参与项目开发等。

-requirements.txt

这个文件主要用来记录项目所需的库，方便在不同环境中快速搭建项目。

-.cursorrules

这个文件是对于特定项目的说明，每开发一个项目，我们都可以在项目文件根目录建一个.cursorrules文件，并向 Cursor提供这个项目的具体指令。这部分工作可以交给cursor chat自己完成，我们只需要注意格式就可以了。

像拼乐高一样实现产品开发功能并进行逐步完善和迭代。

玩乐高的时候，我们需要一点一点的去按照手册去完善功能。乐高的每一个项目是通过不同的模块组装起来的。而且每个项目会有单独的袋子装好每一个模块需要的所有积木。这个过程可能充满各种痛苦，但也充满了挑战的乐趣。同时，也是我们对项目管理认知增进的一个过程。

以“土星五号运载火箭”乐高为例，我们需要做的就是按照手册的指导，一步一步从火箭底座开始搭建。完成第一个模块，再开始第二个模块。以此类推，使用Cursor进行项目开发的过程也是同理的，在面临复杂项目的时候，为了避免陷入无限的纠错状态、或者新生成的最新代码覆盖过去的代码，我们就要引入Git管理代码。

养成良好的git提交习惯

4.1 编程初学者为什么一定要养成Git的提交习惯？

在编程中，管理代码的不同版本非常重要，而Git是一个非常有用的工具。它就像是一个智能的“时间机器”，可以帮我们记录每次修改，随时回到之前的版本。比如，我们前期使用cursor过程中经常遇到的一个问题是：项目开发开发着，就被cursor给带跑了。由于cursor不断的在帮我们生成新代码，从而覆盖之前的代码，因此网页的样式也会不断地发生变化，可能跟我们最初的预期越来越远，以至于项目变得不可控！就像写论文一样，如果你改了很多次，突然想找回最初的版本，Git就能帮你轻松实现这一点。

4.2 Git是目前代码版本控制系统，主要功能如下：

  • 跟踪代码变化： 记录每次代码修改，方便回溯。
  • 创建分支： 在不影响主线代码的情况下，尝试新功能或修复bug。
  • 合并代码： 将不同分支的代码合并。
  • 回滚到任意版本： 随时恢复到之前的版本。

4.3 如何使用好git？

  • 要养成多频次提交git的习惯，可以在终端通过git指令提交。也可以在电脑上下载github desktop，在上面进行提交。每次完成一个功能或修复一个bug，就提交一次代码。
  • 写好提交信息： 详细描述每次提交的修改内容，方便以后查找。
  • 善用分支： 为不同的功能或修复创建不同的分支，保持主线代码的稳定。

4.4 Git的具体操作方法（以终端操作为例）

1.Git回滚：

  • git log # 查看提交历史
  • git reset --hard # 回滚到指定提交

2.Git分支管理：

  • git branch # 查看分支
  • git checkout -b # 创建并切换到新分支
  • git merge # 合并分支

4.5 Git提交代码的一般流程：

  • 修改代码： 实现新的功能或修复bug。
  • 添加至暂存区： 使用git add <文件名>或git add .将修改过的文件添加到暂存区。
  • 提交到本地仓库： 使用git commit -m "提交信息"将暂存区的内容提交到本地仓库。提交信息要清晰明了，描述本次提交的修改内容。

举例：

# 修改了index.html文件
git add index.html
git commit -m "修复了首页的样式问题"

1.编程逻辑是什么

编程就像设计一个玩具火车轨道，你要先铺好路线（写代码），让火车（程序）按照轨道开；如果轨道断了或铺错了方向，火车就会“出轨”或开到奇怪的地方。

1.1 如果用相对严谨的文字来表示，我们可以这样理解编程逻辑的基本原则：

  • 由一个一个的指令
  • 按照一定的逻辑规则编排而成
  • 核心：在编写程序之前，一定要把自己的业务逻辑，梳理的非常清楚，然后通过Cursor相应的指令，实现就可以了。

举个例子：

通过编程逻辑自动化重复性任务（如用Python的Excel处理工具pandas处理数据）

你收到一份考勤表，想统计：

  1. 每位员工的总打卡次数。
  1. 每位员工是否按时上班（打卡时间是否在9:00之前）。
  1. 把整理好的表格存为新的Excel文件。

实现步骤：

  1. 读取考勤表： 用Python读取原始Excel文件，把数据加载进程序。
  1. 处理数据：
  • 检查每位员工的打卡时间。
  • 标记是否按时上班（打卡时间早于9:00）。
  1. 统计结果：
  • 按员工分组，统计总打卡次数。
  • 计算按时上班的天数和百分比。
  1. 保存新表格： 把处理后的结果保存为新Excel文件，交给HR使用。

代码如下：

import pandas as pd

# 读取Excel文件
data = pd.read_excel("raw_attendance.xlsx")

# 转换打卡时间格式
data['打卡时间'] = pd.to_datetime(data['打卡时间'], format='%H:%M')

# 判断是否按时上班
data['按时上班'] = data['打卡时间'].dt.hour < 9

# 统计每位员工的打卡情况
summary = data.groupby(['员工ID', '姓名']).agg(
    总打卡次数=('日期', 'count'),
    按时上班天数=('按时上班', 'sum')
).reset_index()

# 保存结果
summary.to_excel("processed_attendance.xlsx", index=False)

处理后的表格会告诉你：

  • 每位员工打卡了几次。
  • 按时上班了多少次。
  • 按时上班的比例是多少。

通过python代码的一个实际应用，我们可以比较容易的理解什么是编程逻辑，下面我们来了解一下计算机的基本运作逻辑，将有助于我们从整体上了解编程的整体逻辑。

1.2 计算机的逻辑是什么？

编程语言通过定义清晰的语法和语义，使得程序员能够以逻辑顺序描述问题的步骤，常见的逻辑元素包括：

  • 顺序执行：指令按顺序依次执行。
  • 条件判断：根据不同的条件选择执行路径（例如：if、else语句）。
  • 循环控制：重复执行某些操作，直到满足特定条件（例如：for、while循环）。
  • 函数与模块化：将复杂任务拆解为更小的、可重复使用的部分，增强代码的逻辑清晰度和可维护性。

举个例子：

  • CPU（中央处理器）：是大脑，负责计算。
  • 内存：是笔记本，记录暂时需要用的数据。
  • 硬盘：是仓库，存储长期不变的信息。
  • 程序：是说明书，告诉CPU具体该怎么做。

1.3 什么是二进制？

二进制是计算机的核心，计算机的底层处理和数据存储都是基于二进制（0和1）来实现的。

1.3.1为什么是二进制？

因为计算机内部使用电流的开关状态（开和关）来表示数据，就好像我们每天开关的灯泡💡一样！而二进制正好用两种状态（0和1）来表示这种开关状态。计算机中的每个“比特”代表一个二进制位（bit），而多个比特可以组合成更大的数据单位，如字节（byte）等。

1.3.2计算机如何使用二进制？

  • 数据存储：计算机内存和硬盘等存储设备将所有数据（文字、图像、声音等）转化为二进制数字进行存储。
  • 运算处理：所有的数学运算、逻辑运算（如加法、乘法、比较等）最终都会转换成二进制操作，由计算机的中央处理单元（CPU）执行。

以上就是关于编程逻辑的基础内容了。

2.2 与Chat对话的乐趣

前两天在跟Cursor对话的过程中发现了一个非常有意思的现象，或许也可以侧面来感受一下AI的思维方式。我当时正在做一款论坛，涉及到相对复杂的界面布局结构和各种小功能。最开始我是通过纯口语要求Cursor帮我实现功能的，

举例：

“请帮我在页面右上角开发一个个人主页的折叠功能”，接下来cursor就开始自由发挥了。然后你会发现，它会不断地生成新的代码，来匹配你的需求。不过，就算尝试多少次，它呈现出来的效果依然不是不想要的。

当然，我们还可以用另外一种方法来实现比较困难的需求。当我们想实现某个功能或样式，却又不知道该如何表达的时候，可以参考一下类似的网站或平台的样式，并截图，然后发给cursor，让它帮忙总结这个页面的布局效果是如何实现的。然后再让它整理出具体的实现要求。最后再把要求提交给cursor，基本就可以实现我们想要的功能了。

四、方法论

1.思路方法

在使用 Cursor AI 编程工具 之前，理清思路和制定框架是非常重要的，这样可以帮助你更高效地实现目标。以下是一个简洁的思路框架，帮助你明确开发过程中的各个阶段，确保逻辑清晰、目标明确。

1.1 明确目标和需求

  • 目标：首先我们清楚自己想通过 Cursor 实现什么功能，最终达成什么目标？比如是开发一个网站、一个小工具、一个App，小程序、还是一个桌面应用、或者是插件。
  • 需求：列出所有的功能需求，明确每一项功能如何通过 Cursor AI 实现。这可以帮助我们在编写代码时保持聚焦，避免走弯路。

举例：

  • 如果想要开发一个实时爬取小红书的AI工具，需求可能包括：
  • 获取关键词“AI”的最新帖子。
  • 提取帖子标题、笔记内容、作者、正文、点赞、转发、评论等信息。
  • 保存数据、解析数据、可视化展示等。

1.2 设计框架与结构

  • 系统结构：思考整体系统的架构，比如：
  • 前端（用户界面）和后端（数据处理、数据库等）分开处理吗？
  • 如果是爬虫类项目，爬取、存储和展示的过程如何分配？
  • 如果是一个AI应用，数据输入和输出的结构是怎样的？
  • 模块划分：将项目拆解为多个模块或功能模块。每个模块负责独立的功能，便于开发和后期维护。

举例：

  • 爬虫工具：
  1. 数据抓取模块：负责爬取小红书上的数据。
  1. 数据处理模块：清洗和格式化数据。
  1. 展示模块：将数据通过UI展示给用户。
  • AI应用：
  1. 数据输入模块：获取用户输入或处理数据。
  1. AI处理模块：使用机器学习模型进行数据分析或预测。
  1. 输出模块：根据分析结果生成反馈或报告。

1.3 技术选型与工具选择

  • 编程语言：根据需求选择合适的语言。例如，爬虫可能使用 Python，AI可能使用 Python 或其他适合的语言。
  • 开发框架：选择适合项目的开发框架。如果是爬虫，可能用 Scrapy 或 BeautifulSoup；如果是前端，比较常使用 的框架包括React 或 Vue.js；后端可能选择 Flask、Django 或 Node.js 等。

1.4 设定工作流程

  • 规划开发步骤：把项目分解成多个小任务，设定每个阶段的目标和时间节点。比如：
  1. 阶段1：需求分析与功能规划。
  1. 阶段2：数据采集模块开发。
  1. 阶段3：数据处理模块开发。
  1. 阶段4：前端展示界面开发。
  1. 阶段5：测试与优化。
  • 测试与反馈：开发过程中，确保每个功能模块都经过测试，及时解决BUG和优化性能，确保系统整体的稳定性。

1.5 数据和用户交互设计

  • 数据流动：明确数据从哪里来（输入）、怎么处理（数据处理逻辑），最终呈现给用户的形式是什么。
  • 用户体验：考虑到用户操作的流畅性、界面的易用性。是否考虑良好的用户体验和交互的效果！

举例：

  • 如果我想开发一款爬虫工具，需要考虑如何展示获取的数据，例如通过表格、图表或卡片样式。是通过可视化的工具还是通过本地数据库、excel来实现？这部分需要有个大致的了解，然后将其放在自定义的规则文件或项目架构文件里。

1.6 启动开发与迭代

  • 开始开发：从最基本的功能开始实现，逐步迭代，测试每个功能的效果，确保稳定后再增加新功能。这个过程中顺便学习git的使用方法、以及虚拟环境的作用。
  • 代码优化：在开发的过程中不断优化代码的可读性，比如慢慢养成为代码进行注释的习惯，方便后续维护和查找相关代码。

1.7 部署与上线

  • 测试：想一想是否需要进行测试？如何测试？通过什么平台和工具？
  • 准备部署发布：开发完成后，考虑如何将项目部署到线上，例如将爬虫脚本部署在服务器上，或将AI应用部署到云端。
  • 发布：首先要了解服务器、域名、解析、IP等这些基础知识，然后，先找找是否有免费的部署平台，比如Vercel、Cloudflare等；接下来要学习不同应用商店的上传要求。
