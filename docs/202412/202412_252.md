#### **开发心得分享：**

#### 开发心得分享：

1.  **前期规划与构想。** 在项目初期，做好功能设计和构想十分重要。充分利用 .cursorrules 文件，将代码规则和技术方案记录下来，让他随着开发进度及时更新详细的 README 文件和关键代码注释。**小提示** ：Cursor 有时可能忘记之前的工作，这时让它读取 README 文件了解项目进度是个好办法。

2.  **善用 GitHub 管理项目。** 每完成一个满意的设计，要及时提交并推送到 GitHub。Cursor 在修复 bug 时有时会改动过多，甚至破坏已有成果。如果发现问题，可以直接回滚到之前的版本，避免无谓的浪费时间。

3.  **避免完全依赖 Cursor，学会引导。** 避免直接将后台报的错误抛给 Cursor ，容易让他陷入死循环。在遇到 bug 时，引导 cursor 从分析代码结构、前后端接口、程序架构上解决问题。如果你能大概判断出问题所在，提供清晰的方向给 Cursor，可以显著提高问题解决的效率。

4.  **向 ChatGPT 请教，解决上线难题。** 在微信小程序的发布审核中，我遇到了“涉及信息资讯”“深度合成技术”“算命、占卜、风水”等问题，多次被驳回。后来向 ChatGPT 请教，根据它的建议调整了程序内容，并在功能描述中优化了表述，最终顺利通过审核。

5.  **用自动化脚本提升应用性能。** 我的程序调用智谱 API 生成星座运势，用户实时查询时会较慢。为此，我用 Cursor 写了一个定时任务脚本，每天零点自动生成星座运势并存入数据库，用户查询时直接访问数据库，大大提高了应用速度。

你应始终遵循以下原则: ##第一步:项目初始化
-当用户提出任何需求时，首先浏览项目根目录下的 README.md 文件和所有代码文档，理解项目目标、架构和实现方式。 -如果还没有 README 文件，创建一个。
这个文件将作为项目功能的说明书和你对项目内容的规划。
-在 README.md 中清晰描述所有文件的用途、布局结构、样式说明等，确保用户可以轻松理解程序的结构和样式。 ##第二步:需求分析和开发
###理解用户需求时: -充分理解用户需求，站在用户角度思考。 -作为产品经理，分析需求是否存在缺漏， 与用户讨论并完善需求。
-选择最简单的解决方案来满足用户 ##编写代码时: -使用 uniapp 前端框架开发，后端使用 python，数据库使用 mysql。 -确保代码结构清晰。
-采用响应式设计，确保在不同设备上都能良好显示。 -使用 CSS Flexbox 和 Grid 布局实现页面结构。 -都要添加详细的中文注释。
-确保代码符合 W3C 标准规范。 -优化图片和媒体资源的加载。 ###解决问题时: -全面阅读相关相关代码，理解页面结构和样式。
-分析显示异常的原因，提出解决问题的思路。 -与用户进行多次交互，根据反馈调整程序设计。 ##第三步:项目总结和优化-
完成任务后，反思完成步骤，思考项目可能存在的问题和改进方式。 -每完成一部都要更新 README.md
文件，包括页面结构说明、优化建议、系统架构、像项目完成进度。 -优化程序程序页面加载性能，包括 CSS 压缩和图片优化。 -确保小程序跨平台都能正常显示。
在整个过程中，确保使用最新的技术开发最佳实践。
